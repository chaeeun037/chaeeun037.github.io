## 1. 컨셉 정의

**이 블로그는 개발자의 탐험 기록 보관소다.** 딥다이브 아티클은 잠수 기록, 트러블슈팅은 트레일에서 길을 찾은 기록, 회고는 베이스캠프 모닥불 앞의 일지. 운영자의 정체성(깊이 파고드는 개발자 × 바다 수영·백패킹·캠핑을 즐기는 사람)이 장식이 아닌 은유로 작동한다.

- **브랜드 키워드**: 깊이(depth), 탐험(expedition), 기록(log), 따뜻한 자연(warm nature), 픽셀의 절제(restrained pixel)
- **무드 레퍼런스**: 데이브 더 다이버의 "픽셀 월드 + 모던 UI" 공존 구조, 따뜻한 톤의 자연 픽셀 아트. **참조는 무드까지만 — 스프라이트·캐릭터·고유 디자인 차용 금지** (8장 안티골)
- **핵심 통찰**: "딥다이브 아티클"이라는 콘텐츠 전략 1순위를 문자 그대로 시각화한다. 테마와 콘텐츠 전략이 한 몸.

## 3. 디자인 원칙

1. **월드는 픽셀, 텍스트는 모던.** 픽셀 요소는 크롬(chrome)에만 존재하고 글 본문 영역은 성역이다. 크롬 = 히어로, About, 404, 빈 상태, 뱃지, 푸터, 마이크로 인터랙션. 본문 = 아티클 텍스트 컨테이너 내부 전체.
2. **Zone 시스템** (D1 구체화)
   - **Zone A (픽셀 씬 허용)**: 히어로, About, 404, 빈 상태(검색 결과 없음 등), 푸터 소품
   - **Zone B (액센트만)**: 글 목록, 글 상세, 워크벤치 — 색, 작은 뱃지, 미세한 보더 처리까지만. 일러스트·씬 금지
3. **접근성 비협상 항목**: 본문 텍스트 대비 WCAG AA 이상. 모든 모션은 `prefers-reduced-motion: reduce` 존중. 픽셀 폰트는 장식 한정 — 본문·내비게이션 라벨에 사용 금지.
4. **성능 예산**: 테마 에셋 총량(이미지+폰트) 초기 로드 200KB 이하 목표. 픽셀 아트는 원본이 작아 유리하나, 씬 이미지는 lazy load.
5. **채용담당자 30초 시나리오 우선.** 테마는 첫인상을 돕는 도구지 주인공이 아니다. 대표글 도달 경로를 테마가 1초라도 늦추면 그 요소는 뺀다.

## 4. 비주얼 시스템

### 4.1 팔레트 (v0 제안 — 시안 후 확정)

컨셉: 바다의 깊이 그라디언트 + 캠프파이어 웜 액센트. 픽셀 에셋용 제한 팔레트와 UI용 시맨틱 컬러를 분리한다.

| 역할 | 제안 | 비고 |
|------|------|------|
| 표층수 | `#4ECDC4` | 히어로 상단, 라이트 액센트 |
| 중층수 | `#1A6B8A` | 주 브랜드 컬러 후보 |
| 심해 | `#0D2B45` | 다크 배경, 푸터 |
| 캠프파이어 | `#FFB347` | CTA, 하이라이트, 대표글 마킹 |
| 모래/트레일 | `#E8D5B7` | 라이트 배경 보조 |
| 본문 텍스트 | 중립 그레이 스케일 | 픽셀 팔레트와 독립, AA 대비 우선 |

픽셀 에셋은 위 계열 기반 **32색 이하 고정 팔레트**로 제한(에셋 간 일관성 장치). 확정 팔레트는 제미나이 첫 시안 배치에서 가장 좋은 결과물의 색을 역추출해 정한다.

### 4.2 타이포그래피

- **본문·UI**: Pretendard (또는 현 개발 중 폰트 유지)
- **장식·제목 포인트**: 한글 픽셀 폰트 — Galmuri 계열 우선 검토(무료, 다양한 굵기). 히어로 타이틀, 뱃지 라벨, 404 문구 한정
- 픽셀 폰트 사용 상한: 한 화면에 2곳 이하

### 4.3 픽셀 렌더링 규칙

- 모든 픽셀 이미지: `image-rendering: pixelated` + **정수 배율만** (1x, 2x, 3x — 1.5배 금지)
- 그리드 규격: 아이콘 16×16 / 뱃지·소품 32×32 / 아바타 64×64 / 씬 320×180 기준(정수 배율 확대)
- 안티앨리어싱 없는 하드 엣지가 원칙 — 후처리 파이프라인(6장)에서 강제

## 6. 에셋 파이프라인

**분업 원칙: 제미나이(나노바나나)가 그리고, 클로드가 다듬는다.**

### 6.1 생성 (제미나이)

- 프롬프트 골격: `warm-toned pixel art, [대상], 32-color limited palette, flat solid colors, hard edges, no anti-aliasing, simple shapes, [배경 지시], game asset style`
- **일관성 장치**: 첫 배치에서 베스트 결과물 1장을 선정 → 이후 모든 생성에 레퍼런스 이미지로 첨부 + 팔레트 hex 명시
- **IP 거리두기**: 특정 게임명 프롬프트 금지. 특정 게임 캐릭터와 유사한 결과물은 폐기
- 첫 배치 목록: 아바타(다이버/백패커 2안), 히어로 씬(수면→수중), 캠프파이어, 텐트, 랜턴, 산호/물고기 소품 2~3종

### 6.2 후처리 (클로드코드 스크립트)

`scripts/pixel-pipeline/` 에 구축:

1. **다운스케일**: 생성 원본(고해상 "가짜 픽셀")을 목표 그리드(4.3)로 최근접 이웃(nearest-neighbor) 축소
2. **팔레트 양자화**: 공용 32색 팔레트로 강제 매핑 (에셋 간 색 통일)
3. **엣지 정리**: 알파 채널 하드 컷 (반투명 픽셀 제거)
4. 도구: sharp 또는 ImageMagick. 입력 `assets-raw/` → 출력 `public/pixel/`

### 6.3 규격·네이밍

- 포맷: PNG (투명 필요시) / 아이콘류는 SVG 재작(클로드) 우선 검토
- 네이밍: `{zone}-{대상}-{크기}.png` 예: `hero-underwater-320x180.png`, `about-avatar-64.png`

## 7. 적용 페이즈 & 킬 스위치

- **선행 조건: PRD MS3(대표글 큐레이션 + 첫 글 발행)까지 클린 완주 후 착수.** 그 전에는 4.1 팔레트의 시맨틱 컬러 변수만 선반영 가능(나중에 테마 입힐 때 리스킨 비용 최소화).
- Phase T1: 팔레트·타이포 변수 적용 + 수심 뱃지 + 대표글 마킹 (P1 중 Zone B)
- Phase T2: 히어로 씬 + About 아바타/프로필 (P1 중 Zone A) — 에셋 파이프라인 구축 포함
- Phase T3: 404, 빈 상태, 진행바 (P2)
- **킬 스위치**: 테마 작업이 2주 연속 글 작성 시간을 잠식하면 즉시 중단하고 집필 사이클로 복귀. 판단 기준은 단순하다 — *2027년 2월에 평가받는 것은 테마가 아니라 대표글 3~5개다.*
- 각 Phase 완료 시 빌딩 로그 글감 후보: "픽셀 월드와 모던 UI의 공존 — 게임에서 배운 블로그 디자인", "AI 생성 픽셀 아트를 진짜 픽셀 아트로 만드는 후처리 파이프라인"
