## 1. 원정 기록 (경력 하이라이트 · 우선순위 순)

> 각 항목: **타이틀(성과 중심)** + 부연(어떻게/왜) + 핵심 수치(verified만). 대표글 동선은 3장 매핑 참조.

### 원정 1 — 모노레포 배포 시간 34.2% 단축

- Docker 레이어 캐싱(의존성 먼저 COPY)과 BuildKit registry cache를 붙여, 소스만 바뀌어도 매번 의존성을 재설치하던 배포를 재설계.
- **수치: 배포 시간 약 20분 → 13분대(-34.2%)**(verified — 2026-2Q KPI 평가 시트로 확정). *2026 Q2*

### 원정 2 — 8개월 묵은 사파리 채팅 WebSocket 난제 해결

- 다른 개발자들도 오래 못 풀던 이슈를, 프론트 범위를 넘어 인프라(CORS 등) 영역까지 원인을 추적해 마무리.
- **수치: 약 8개월 미해결 이슈 해결**(verified). *2024 하반기 · 사파리 채팅 안정성 확보*

### 원정 3 — 사파리에서 차단되던 백필 광고를 인프라 레벨에서 복구

- 서드파티 광고 SDK를 도입하며, 보안 헤더(X-Frame-Options 제거·CSP 수립)로 사파리에서 차단되던 백필 광고를 정상 노출로 복구. 가상 렌더링에서 중복 수신되던 노출을 캐싱으로 집계 정합성까지 확보.
- **성과: 광고 CPV 목표 달성**(달성 여부는 verified — 2026-1Q 평가자 확인) **+ 노출 차단 해소·집계 정합성 확보**(트러블슈팅 verified). *2026 Q1 / 구체 수치(월 건수·초과분)는 사업 지표라 전부 비노출*

### 원정 4 — 대량 푸시로 죽던 서버 부하를 캐싱으로 근본 해소

- 앱 푸시 대량 발송 시 과호출되던 로그 수집 API를 24h TTL 캐시 + inFlight 중복 제거 + 로그 함수 싱글톤 수렴으로 재설계.
- **수치: 배포 후 캐시 적중률 92.5%**(verified). *2025 Q4*

### 원정 5 — 흩어진 팀 컨벤션을 실행 가능한 PR 게이트로

- 리뷰에서 반복 지적되던 팀 컨벤션 ~40개를 PR 직전 자동으로 도는 프리플라이트 린터로 만들고, 기계적 검사와 LLM 검사를 성질별로 분리해 비용을 튜닝.
- **수치: LLM 컨벤션 검사 76초 → 5초**(verified). *2026 Q2 · 개발 워크플로/툴링*

> **교체 후보(정성 성과 — 수치보다 판단·설계력 신호):** 필요 시 원정 5와 교체 가능
>
> - **신청서 거대 파일 안전 분리**: 135+ 훅 거대 파일을 동작 보존(Strangler) 원칙으로 분리, 참조 안정성으로 불필요한 리렌더 차단 (2026 Q2, verified·정성)
> - **에러 계약 관측성**: `[object Object]` 노출을, 프론트 추측 대신 BE 레포 2곳 정적 분석으로 잡고 재발 관측 지점 설계·호출부 100+곳 계약 불변 유지 (2026 Q3, verified·정성)

## 2. 장비 슬롯 (기술 스택)

> 현재 초안: React · TypeScript · Next.js · React Query · 테스트(Jest/RTL/MSW)
> 실제 원장 근거로 재정렬 + 인프라/관측 계열을 차별점으로 노출. ~~React Query·테스트 스택 근거 미확인~~ → ✅ 본인 확인 완료(2026-07-19, Q3·Q4 해소 — 4장 참조)

**주력 (핵심 · 상시 사용)**

- React · TypeScript · Next.js (getServerSideProps SSR 분기, Next.js Image 캐싱까지 실무 적용)
- React Query — 실사용 중, 캐싱 정책(staleTime 등) 심화는 학습 지속 중(→ dev-journal react-query 학습 로드맵). 면접 대비 필요.

**실무 검증**

- 모노레포(yarn workspaces / 공용 `_packages`) · Storybook · Feature Toggle·트렁크 기반 개발
- 컴포넌트 설계(discriminated union) · Optimistic Update · 렌더 성능 최적화(useMemo/참조 안정성)

**인프라 · 관측 (프론트 밖까지 — 차별점)**

- Docker · GitHub Actions · AWS ECR · ArgoCD · CloudFront · Nginx (CI/CD 자체 구축 경험)
- Sentry · Datadog · Custom Log/SQL 집계 (운영 에러 선별·배포 전후 데이터 측정)

**AI 개발 워크플로**

- Claude Code 멀티 에이전트 오케스트레이션(개인 하네스) · CodeRabbit 등 AI 코드리뷰 선별 적용

> 조정 근거: 원장 전반에서 React/TS/Next.js·모노레포·SSR·Feature Toggle·인프라(Docker~~ArgoCD~~Nginx)·관측(Sentry/Datadog/Custom Log)이 반복 등장. React Query·Jest/RTL/MSW는 2026-07-19 본인 확인으로 실사용 확정. React Query는 앞으로 계속 쓰며 익힐 예정이라 **주력 유지**하되, 캐싱 정책 심화 학습을 병행(로드맵: dev-journal/momsitter/react-query-learning-plan-2026-07-19.md — 면접 전 필수 소화).
